# Voxtral-max: повышение качества транскрибации

**Дата:** 2026-06-08
**Статус:** дизайн утверждён, готов к плану реализации

## Контекст и проблема

Пользователь сравнил нашу транскрибацию (Dicto/Voxtral) с конкурентом
(speech2text.ru) на одном зашумлённом телефонном разговоре
(`220428_001.MP3`, 6:37, газовое хозяйство). Конкурент выглядит «почти
идеально». Задача — дотянуться по качеству.

### Что показал эмпирический замер (на этом файле)

Прогнали 4 движка и сравнили по тяжёлым местам:

| Тяжёлое место | Конкурент | Voxtral+bias | Whisper-ru (base) | Yandex |
|---|---|---|---|---|
| Фамилия (00:11) | Магулов | Мазунов | Моргунов | — |
| «…недоволен» Мустафин | ✅ | ❌ потерял | ❌ потерял | ≈✅ «мутахин» |
| Наиль (06:21) | ✅ | ❌ «на Илю» | ✅ «Наиле» | ❌ |
| объект (04:35) | ✅ | ✅ (bias) | ✅ | ✅ |
| телефон | точнее | частично | частично | прописью |
| читабельность | хорошая | рваная | **лучшая** | каша (дубли) |
| галлюцинации | нет | **нет** | **есть** (выдуманные имена, «Продолжение следует») | нет |

**Выводы:**

1. **Идеал по зарытой фамилии недостижим** — каждый движок слышит её
   по-своему (Магулов/Мазунов/Моргунов/мутахин). Сигнал физически
   разрушен шумом; это не лечится сменой модели или дообучением.
2. **Реальное преимущество конкурента — читабельность и отсутствие
   развала в кашу**, а не магия по именам. Это мы можем закрыть.
3. **Характеры движков:** Voxtral — надёжный, без галлюцинаций, но
   рваный; Whisper — читабельный, берёт «Наиль», но галлюцинирует в
   тишине (архитектурное, файнтюн не лечит); Yandex — отвергнут
   (диаризация даёт дубли-кашу + дороже ×8).

### Решения, принятые до дизайна

- **Движок:** остаёмся на Voxtral. `voxtral-mini-2602` (Voxtral
  Transcribe 2) — подтверждённо топовая и единственная модель
  транскрибации Voxtral; `voxtral-small-*` эндпоинт
  `/v1/audio/transcriptions` отвергает («Invalid model»).
- **Дообучение модели — не сейчас:** не вытаскивает зарытый звук,
  требует GPU + размеченный датасет + MLOps; преждевременно до
  появления пользователей (YAGNI).
- **`context_bias` работает** (проверено: чинит «объект»,
  «пусконаладочных») — делаем его пользовательским словарём.
- **Whisper-ru self-host** задокументирован как эксперимент на потом
  (нужен GPU + борьба с галлюцинациями), в текущий скоуп не входит.

## Цель

Приблизить читабельность и стабильность к уровню конкурента, оставаясь
на дешёвом Voxtral API, без своего GPU. Метрика успеха: на этом и
похожих файлах текст читается связными абзацами с пунктуацией, имена и
термины из пользовательского словаря распознаются, явный мусор
вычищен — без выдуманного контента.

## Архитектура

Пайплайн в `tasks/transcribe.py`:

```
S3 → [денойз FFmpeg] → Voxtral(context_bias=словарь) → [LLM-постобработка] → save
```

Два новых шага — изолированные сервисы. Таймкоды сегментов сохраняются
на всех этапах (нужны для студии/плеера/SRT).

### Компонент 1: пользовательский словарь (`context_bias`)

- **Модель:** новая колонка `Transcription.vocabulary: str | None`
  (Alembic-миграция).
- **API:** необязательное поле `vocabulary` (строка) в схеме создания
  транскрипции.
- **Провайдер** (`services/voxtral.py`): санитайзинг в формат Voxtral —
  токенизация, отбрасывание элементов с пробелами/запятыми (паттерн
  API `^[^,\s]+$`), максимум 100 элементов. Передаётся как
  `context_bias`. Сигнатура `transcribe(..., context_bias=None)` уже
  добавлена.
- **Frontend:** поле на странице загрузки — «Имена и термины (через
  запятую)» с подсказкой про фамилии/названия/аббревиатуры.
- **Зависимости:** провайдер не знает про модель/БД; получает готовую
  строку от задачи.

### Компонент 2: LLM-постобработка (`services/transcript_postprocess.py`)

- **Вход:** список сегментов Voxtral (`start`, `end`, `text`,
  `speaker`). **Выход:** тот же список с исправленным `text`, **то же
  число сегментов, те же тайминги**.
- **Модель:** `mistral-small-latest` (уже в стеке, см.
  `services/ai_analysis.py`).
- **Промпт (строгий):** исправь пунктуацию, регистр, очевидные ошибки
  распознавания; нормализуй имена/термины по контексту и по
  пользовательскому словарю. ЗАПРЕЩЕНО: добавлять, удалять или менять
  смысл; объединять или дробить сегменты; менять их количество. Верни
  ровно N сегментов по индексам.
- **Защита целостности:** если LLM вернул не N сегментов или сломал
  структуру — откатываемся на исходные сегменты (fail-safe, не роняем
  задачу). Правка строго внутрисегментная → тайминги 1:1.
- **Стоимость:** короткий транскрипт → копейки на запись; добавить в
  оценку себестоимости (сейчас 5.24 ₽/час).
- **Режим:** включено по умолчанию. Тумблер «дословно/verbatim» —
  будущее (YAGNI сейчас).
- **Читабельные абзацы** для DOCX/экспорта — задача слоя
  отображения/экспорта, НЕ разрушает сегментные данные (см. компонент 4).

### Компонент 3: шумоподавление (`services/audio_preprocess.py`)

- Консервативная FFmpeg-цепочка перед ASR:
  `highpass=80, lowpass=8000, afftdn, loudnorm`.
- **Гейт валидации (обязателен в реализации):** прогнать Voxtral на
  `220428_001.MP3` с денойзом и без; оставить шаг, ТОЛЬКО если он
  улучшает (меньше мусора/искажений). Если нет — выпилить из пайплайна.
  Без слепого внедрения.
- **Fail-safe:** отсутствие ffmpeg или ошибка фильтра → используем
  оригинальный аудиофайл, задача не падает. (FFmpeg уже используется
  для видео — переиспользовать обёртку.)

### Компонент 4: диаризация / абзацы

- Сначала выяснить, почему `diarize=true` не вернул `speaker` (sync vs
  batch-эндпоинт / формат ответа Voxtral Transcribe 2).
- **A-итерация:** если настоящая диаризация дёшево не заводится —
  группировка сегментов в абзацы по паузам (порог паузы между `end`
  предыдущего и `start` следующего). Даёт читабельную «разбивку», как у
  конкурента, без имён спикеров. Применяется в слое
  отображения/экспорта.
- Настоящая диаризация (Спикер 1/2) — отдельная задача позже.

## Тестирование

- `transcript_postprocess`: мок Mistral → число сегментов и тайминги
  сохранены, текст подменён; ветка fail-safe (LLM вернул не N сегментов
  → откат на исходные).
- `voxtral`: санитайзинг словаря (отброс пробелов/запятых, лимит 100,
  пустой словарь → параметр не отправляется).
- `audio_preprocess`: команда FFmpeg собирается корректно; отсутствие
  ffmpeg → fallback на оригинал без падения.
- Миграция `vocabulary` применяется (`alembic upgrade head`).
- Бэкенд-тесты на SQLite in-memory (как в `conftest.py`).

## Границы скоупа (что НЕ делаем сейчас)

- Дообучение/файнтюн любой модели.
- Whisper-ru / GigaAM / Yandex как движок (Yandex отвергнут замером).
- Voxtral Small через chat-эндпоинт (нет таймкодов/диаризации).
- Настоящая диаризация спикеров (только абзацы по паузам).
- Тумблер verbatim.
