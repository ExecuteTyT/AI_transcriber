# Dicto — валидационный тест клина + первый реальный пейволл

**Дата:** 2026-06-10
**Статус:** дизайн на ревью
**Автор:** Claude (senior product/growth) + Ислам

---

## 1. Контекст и проблема

Dicto в проде: ~524 пользователя, 607 транскрипций, 374 активных — и **0 оплат**.

Диагностика (по коду и прод-данным) вскрыла три корня:

1. **До пейволла никто не доходит.** Free даёт 180 бонусных минут разово; средний активный юзер тратит ~21 мин (12%). Стена «минуты исчерпаны» почти не срабатывает → нет момента оплаты.
2. **Даже на стене не продаём.** При исчерпании лимита — сухой `403 «лимит исчерпан»`, без экрана тарифов и оплаты.
3. **Реклама покупает не тех.** Текущие объявления («видео в текст», «youtube в текст», «расшифровать голосовое») привели 281 из 374 активных (75%) — любопытствующий one-and-done трафик с видео/соцсетей/ссылок. Эти люди не платят.

Стратегический вывод: Dicto позиционируется как клон speech2text (та же модель, те же цены, тот же commodity «аудио → текст»), конкурируя слабой стороной. Сильная сторона — **AI-разбор** (саммари, тезисы, action items, RAG-чат), которого у speech2text нет, — спрятана.

## 2. Стратегическое решение

Переход из категории «транскрибатор» в «**разбор записанных разговоров**» (conversation intelligence): бесплатно отдаём commodity (текст), деньги берём за отличие (разбор).

Выбран **подход A — reposition-first**: переупаковка существующего AI-слоя + почин монетизации + перенацеливание рекламы, БЕЗ тяжёлой продуктовой стройки. Бот-участник звонка и кросс-поиск по записям — фаза 2, после первых денег.

**Но**: анализ прод-данных показал, что выборка **отравлена текущей рекламой** — по ней нельзя выбрать клин. Поэтому до полной перекройки проводим **мульти-клин валидационный тест**: рынок голосует деньгами на чистом (таргетированном) трафике, затем перекраиваем всё под победителя.

### Зафиксированные решения
- **Клин-кандидаты (3):** разговоры 1:1 (юристы/консультанты/коучи) · контент-создатели (подкасты/YouTube) · обучение/лекции (лекторы/вебинары).
- **Сигнал готовности платить:** реальный пейволл → YooKassa (зарабатываем сразу + сильнейший сигнал).
- **Граница free/Pro:** гейтим AI-разбор. Free = расшифровка + спикеры + 1 пробный разбор; Pro = полный AI-разбор + чат по записи.

## 3. Дизайн теста

### 3.1 Архитектура — три изолированные дорожки

Каждый клин получает свою связку, чтобы сигналы не смешивались:

| Клин | Посадочная | Семантика (примеры) |
|---|---|---|
| **1:1-разговоры** | новая `/dlya-konsultacij` (на базе существующей `RasshifrovkaIntervyu`) | расшифровка консультаций, расшифровка переговоров, протокол консультации юриста, расшифровка звонка с клиентом |
| **Контент-создатели** | адаптируем существующую `DlyaPodkastov` | транскрибация подкаста, субтитры для youtube, текст из выпуска, расшифровка эфира |
| **Обучение/лекции** | адаптируем существующую `DlyaLekcij` | расшифровка вебинара, конспект лекции из аудио, расшифровка урока/занятия |

Каждая дорожка размечена `utm_content=<wedge>` → в Метрике и прод-SQL каждый клин виден отдельно по всей воронке.

### 3.2 Воронка и инструментирование

Единая воронка (одинаковые метки для честного сравнения):

```
клик → регистрация → загрузка (активация) → открыл AI-разбор (дошёл до ценности)
     → упёрся в пейволл → начал оплату → ОПЛАТИЛ
```

Цели Метрики: `registration` (562706351), `upload` (562706449) уже есть. Добавить:
- `analysis_view` — открыл результат/попытался получить разбор (момент ценности),
- `paywall_hit` — увидел экран пейволла,
- `checkout_started` — нажал «оплатить» (редирект в YooKassa),
- `purchase` (562706467) уже есть — фиксирует факт оплаты.

### 3.3 Монетизация — модель «Кошелёк + Pro» (ценность в разборе)

Два типа клиента → два пути оплаты (подписка speech2text ловит только один: регулярных, теряя эпизодическое большинство).

**Экономика для контекста:** себестоимость 0.087 ₽/мин (расшифровка 0.080 + разбор 0.007). 180 бесплатных минут = ~15 ₽ макс / ~1.8 ₽ факт на юзера. Привлечение = 23-80 ₽. Вывод: разоряет не бесплатная расшифровка (копейки), а реклама на неплатящих. Поэтому модель отличается **механикой**, а не ценой.

**Воронка:**
1. Реклама клина → лендинг → регистрация.
2. **Проба (one-time):** первый файл (до ~30-60 мин) расшифровка + **1 бесплатный AI-разбор** — показываем и текст, и разбор (отличие). Заменяет нынешние 180 мин / 5 разборов.
3. Хочет ещё (второй файл / ещё разбор / чат по записи) → **пейволл с двумя путями:**
   - **Кошелёк** — разово пополнить баланс (от ~299 ₽), тратится на расшифровку (~1.5-2 ₽/мин) + разборы. Без подписки. Для эпизодического большинства.
   - **Pro** — подписка (~500 ₽/мес, текущий план `start`): безлимит разборов + чат + хранение. Для регулярных профи.
4. Использование списывает баланс ИЛИ покрывается Pro.

**Границы free/платно:**
- Бесплатно (проба, one-time): 1 файл + 1 разбор.
- Платно: всё сверх пробы — с кошелька или по Pro. Разбор/чат — премиум-ценность, но гейт срабатывает просто «проба исчерпана».
- Минутный бонус 180 → заменяется пробой (правка free-констант в `plans.py`).

**Ценообразование (ориентир, уточняем на ревью):** расшифровка с баланса ~1.5-2 ₽/мин (маржа кратная при себестоимости 0.087), кошелёк от 299 ₽, Pro 500 ₽/мес. Цены НЕ выносим в витрину как «дешевле speech2text».

**Сигнал для теста:** оба пути ведут в YooKassa; считаем `checkout_started` и `purchase` отдельно по пути (кошелёк/Pro) и по клину.

> **Что есть / чего нет:** Pro-подписка через YooKassa уже работает (`create_payment`, webhook, `activate_subscription`). **Кошелёк (баланс + пополнение + списание) — НОВЫЙ модуль**, его нет в коде. См. вопрос о фазировании ниже (§3.5).

> **Открытый вопрос (на ревью):** гейтить ли экспорт (DOCX/SRT)? Коммит `c8bcc55` сознательно открыл DOCX на всех тарифах. По умолчанию экспорт **оставляем бесплатным** (входит в пробу и далее), гейтим ценность через разбор/чат + лимит пробы. Подтвердить.

### 3.5 Фазирование сборки кошелька — РЕШЕНО: вариант (а)

**Решение (2026-06-10): минимальный кошелёк + Pro в тесте.** Собираем баланс у `User` + пополнение через YooKassa (credit на сумму) + списание при использовании + пейволл с 2 путями. БЕЗ истории транзакций / чеков-ledger / авто-возвратов — это после теста. Оба пути оплаты доступны в тесте → не недооцениваем эпизодическое большинство.

Отклонено: (б) Pro-only — недооценит большинство, не берущее подписку; (в) полный кошелёк — слишком много для 3 недель.

### 3.4 Бюджет, срок, правило решения

- **Срок:** 3 недели; дата решения фиксируется заранее.
- **Бюджет:** ~15 000 ₽ суммарно (≈5 000/нед), поровну между 3 клинами; перелив в лидера запрещён до даты решения.
- **Текущая «видео/youtube» кампания (710222862) — на паузу** на время теста (иначе зальёт мусором и испортит чистоту сравнения).
- **Правило победителя (pre-registered):** победитель = клин с минимальным **CPA до `purchase`**; при отсутствии оплат — по минимальному cost-per-`checkout_started`, затем по связке активация×возврат. Минимум ~50 регистраций в клине, иначе статус «недобор статистики» (не засчитывается ни как победа, ни как провал).
- **Kill-switch:** клин с CPL > 200 ₽ и 0 активаций к концу недели 1 — пауза; бюджет НЕ переливаем (замораживаем).
- **Риск объёма:** у 1:1-разговоров мало запросов — может не набрать показов. Это сам по себе сигнал (узкий рынок), фиксируем, не считаем провалом инструмента.

## 4. Что строим (компоненты)

1. **Frontend — посадочные:** 1 новая (`/dlya-konsultacij`) + адаптация 2 существующих (`DlyaPodkastov`, `DlyaLekcij`) под конверсию (оффер клина, пример результата-разбора, CTA на регистрацию). UTM-проброс.
2. **Frontend — пейволл с 2 путями:** экран «Кошелёк / Pro» в момент ценности (`Transcription.tsx` / после пробы) → выбор пути → YooKassa. Копирайт + пример результата под клин.
3. **Backend — монетизация (кошелёк = НОВОЕ):**
   - правка free-констант в `plans.py` (проба = 1 файл + 1 разбор вместо 180 мин / 5 разборов);
   - **кошелёк:** поле баланса у `User`, пополнение через YooKassa (credit произвольной суммы — расширение `create_payment`/webhook на тип «top-up»), списание при использовании (минуты + разбор);
   - гейтинг пробы отдаёт структурированный ответ с маркером «нужен баланс или Pro» (вместо общего `403`);
   - проверка end-to-end в проде (Pro уже работает — подтверждено первым реальным платежом 10.06).
4. **Metrika:** новые цели (`analysis_view`, `paywall_hit`, `checkout_started`); проверка привязки и срабатывания; UTM-схема per-wedge.
5. **Директ (API):** пауза текущей кампании; 3 новые группы (семантика + по 3-5 объявлений + UTM) под клины; единый счётчик/цель; дата решения.
6. **Аналитика:** дашборд/набор SQL (прод) + Метрика-запросы по воронке в разрезе `utm_content`; readout на 3-й неделе по pre-registered правилу.

## 5. Критерии успеха

- **Первичный:** ≥1 клин даёт реальные оплаты при вменяемом CPA (целевой CPL по регистрации был 80 ₽; для оплаты — считаем юнит-экономику отдельно, маржа платных тарифов ≥75%).
- **Вторичный (если оплат мало за 3 недели):** чёткий ранг клинов по `checkout_started` + активация×возврат — достаточно, чтобы выбрать, подо что перекраивать.
- **Решение после теста:** коммитим полную перекройку (сайт/тарифы/реклама — куски №2-5 из исходной декомпозиции) под клин-победитель.

## 6. Вне рамок (явно НЕ делаем сейчас)

- Полная перекройка сайта/тарифов под победителя (после теста).
- Бот-участник звонка (Zoom/Meet/Телемост) — фаза 2.
- Кросс-поиск по всем записям пользователя (RAG across recordings) — фаза 2.
- Полноценный кошелёк (история транзакций, чеки-ledger, авто-возвраты) — после теста; в тесте минимальный (баланс + пополнение + списание).
- Мультиюзер/командные функции.

## 7. Риски и допущения

- **N костяка мал** (~43 юзера с ≥3 записями) → тест и нужен, чтобы получить чистые данные.
- **Узость 1:1-сегмента** по рекламному объёму (см. kill-switch).
- **Правка free бьёт по всем юзерам, не только тест-когорте.** Допустимо: подписчиков ещё нет, grandfather не нужен (см. memory `feedback_no-grandfather-when-no-users`). Видео-кампания на паузе, так что верх воронки в тесте — только целевой.
- **YooKassa в проде** должна реально пробивать чек — проверяется тестовой оплатой до старта.
- **152-ФЗ:** запись чужих разговоров — privacy-чувствительно; для 1:1-клина авто-удаление аудио превращается в аргумент продажи, не риск.
