# Дизайн: переработка навигации, перелинковки и секций главной (SEO + конверсия)

## Context

dicto.pro — лендинг сервиса транскрибации. После добавления 17 новых SEO-страниц (всего 34 лендинга) обнаружилось:

1. **Навигация рассинхронизирована.** В шапке 4 пункта: «Возможности», «Кому» — скролл к секциям главной; «Тарифы» — скролл к мини-секции, хотя есть полноценная индексируемая `/pricing`; «Блог» — отдельная страница. Пользователь заметил несогласованность.
2. **34 SEO-лендинга почти не перелинкованы.** Футер показывает 4-5 ссылок. Краулеры плохо находят новые страницы — индексация замедлена, ссылочный вес не распределяется.
3. **Главная не оптимизирована под конверсию и head-запросы.** Нет акцента на free-оффере у точек решения, нет текстового SEO-блока под ВЧ-ключи.

**Цели (подтверждены пользователем):** больше SEO-трафика, понятная навигация, выше конверсия.

**Решения, отклонённые сознательно:**
- Дробить «Возможности»/«Кому» в отдельные страницы — нет поискового спроса на такие запросы, чистая работа ради работы. Якорный скролл остаётся.
- Выдуманные отзывы/соц-доказательство — реальных отзывов нет, выдумывать нельзя (риск доверия и закона).
- Dropdown-мегаменю в шапке — лишняя сложность и мобильное трение; перелинковку даёт mega-футер.

## Часть 1 — Навигация (консистентность)

Верхнее меню остаётся плоским (4 пункта), меняется только поведение «Тарифы»:

| Пункт | Было | Станет |
|---|---|---|
| Возможности | scroll → `#features` | без изменений |
| Кому | scroll → `#use-cases` | без изменений |
| **Тарифы** | scroll → `#pricing` | **`<Link to="/pricing">`** (полная индексируемая страница) |
| Блог | `<Link to="/blog">` | без изменений |

Изменения в `frontend/src/pages/Landing.tsx`: десктоп-меню (~стр. 294) и мобильное меню (~стр. 339) — заменить `<button onClick={scrollToSection("pricing")}>Тарифы</button>` на `<Link to="/pricing">Тарифы</Link>`.

*Обоснование:* `/pricing` богаче мини-секции, индексируется, уже в sitemap и футере — единообразие. Мини-секция тарифов на главной остаётся (для тех, кто скроллит), но навигация ведёт на полную страницу.

*Эффект:* с каждой из 34+ страниц — плотная перелинковка на остальные → ускоряет обход и индексацию, распределяет вес.

## Часть 3 — Конверсия + on-page SEO

### 3.1 CTA / оффер
- У решающих CTA (мини-секция тарифов на главной, финальный CTA) — единая микрокопия-реассуранс под кнопкой: «180 минут бесплатно · без карты».
- Главная кнопка — единое слово действия («Попробовать бесплатно») во всех точках.

### 3.2 SEO-текстовый блок внизу главной
Новая секция (H2 + 2-3 абзаца) с ВЧ-ключами: «транскрибация», «расшифровка аудио», «аудио в текст», «перевод аудио в текст». Цель — чтобы `/` ранжировалась по head-запросам. Редакционная вёрстка, не «портянка». Размещается над секцией «По теме»/футером.

### 3.3 UX-доверие (по практикам, без выдумок)
Вынести реальные факты на видное место у точек решения: «файлы удаляются», «152-ФЗ, хранение в РФ», «без карты». Использовать существующие формулировки из FAQ/SEO-страниц — ничего не выдумываем.

- `npx tsc --noEmit` — без ошибок типов.
- `npm run dev` → проверить: «Тарифы» в шапке ведёт на `/pricing`; mega-футер виден на главной, SEO-странице и в блоге; все ссылки футера ведут на существующие роуты (нет 404); секция «По теме» рендерится; SEO-блок и CTA-микрокопия на месте.
