- Рост ~3.5x за 2 года — рынок в фазе активного роста
- Драйверы: удалёнка, подкасты, онлайн-образование, нейросети

## 2. Целевая аудитория (сегменты)

### Сегмент A: Контент-мейкеры (30%)
- **Кто**: Блогеры, подкастеры, журналисты
- **Потребность**: Текст из подкастов/интервью для статей, субтитры для YouTube
- **Запросы**: "субтитры для видео", "текст из подкаста", "транскрибация youtube"
- **Боль**: Дорого, долго вручную, нужны таймкоды
- **Тариф**: Start/Pro

### Сегмент B: Бизнес-пользователи (35%)
- **Кто**: Менеджеры, HR, аналитики
- **Потребность**: Протоколы совещаний, расшифровка собеседований, action items
- **Запросы**: "расшифровка совещания", "протокол встречи из аудио", "ии помощник для встреч"
- **Боль**: Нет времени вести заметки, теряются договорённости
- **Тариф**: Start/Pro (ценность саммари и action items)

### Сегмент C: Студенты/учёные (20%)
- **Кто**: Студенты, аспиранты, исследователи
- **Потребность**: Расшифровка лекций, интервью для диссертаций
- **Запросы**: "расшифровка лекции бесплатно", "аудио в текст нейросеть бесплатно"
- **Боль**: Бюджет, нужен русский язык
- **Тариф**: Free → Start при росте объёмов

### Сегмент D: Юристы/медики (15%)
- **Кто**: Адвокаты, врачи, нотариусы
- **Потребность**: Расшифровка заседаний, диктовок, приёмов
- **Запросы**: "расшифровка судебного заседания", "диктофон в текст"
- **Боль**: Точность критична, нужна приватность
- **Тариф**: Pro

## 4. SEO-стратегия

### 4.1 Структура сайта (информационная архитектура)
```
/                          → Главная (SPA app для авторизованных)
/audio-v-tekst             → SEO-лендинг: транскрибация аудио в текст
/video-v-tekst             → SEO-лендинг: перевод видео в текст
/nejroset-transkribaciya   → SEO-лендинг: нейросеть для транскрибации
/rasshifrovka-audio        → SEO-лендинг: расшифровка аудио
/rasshifrovka-golosovyh    → SEO-лендинг: расшифровка голосовых
/protokol-soveshchaniya    → SEO-лендинг: протокол совещания из аудио
/pricing                   → Тарифы (индексируемая)
/login                     → Вход
/register                  → Регистрация
```

### 4.2 Принципы SEO-лендингов
1. **Статический HTML** (серверный рендеринг или prerender) для индексации
2. **H1 = основной запрос кластера** (один на страницу)
3. **Schema.org SoftwareApplication** для rich snippets
4. **FAQ-блок** с Schema.org FAQPage — расширенные сниппеты
5. **CTA → регистрация** в каждом блоке
6. **Внутренняя перелинковка** между лендингами
7. **Хлебные крошки** с Schema.org BreadcrumbList

### 4.3 Контентная стратегия
- **Каждый лендинг**: 1500-3000 слов, H1-H2-H3 иерархия, LSI-ключи
- **Блог** (фаза 2): "Как расшифровать подкаст", "Лучшие нейросети для транскрибации"
- **Внешние площадки**: Habr, VC.ru, DTF — обзорные статьи с ссылками
- **Отзывы**: iRecommend, Отзовик — UGC для ссылочной массы

### 4.4 Техническое SEO
- robots.txt: разрешить индексацию лендингов, закрыть /api/
- sitemap.xml: все лендинги + pricing
- canonical URL на каждой странице
- Open Graph + Twitter Cards
- Скорость: Lighthouse > 90 (статический HTML)
- Mobile-first (все лендинги адаптивны)
- hreflang="ru" (фаза 1 — только русский)

## 5. Аудит дизайна и UX-рекомендации

### Проблемы текущего дизайна
1. **index.html**: нет description, OG-тегов, canonical, favicon
2. **Нет SEO-лендингов**: весь контент за авторизацией — поисковики ничего не видят
3. **SPA без SSR**: контент не индексируется (React рендерится на клиенте)
4. **Нет 404 страницы**
5. **Accessibility**: нет label для форм, нет aria-атрибутов
6. **Sidebar**: нет активного состояния для текущей страницы
7. **Pricing**: доступен только авторизованным — должен быть публичным

### Что нужно сделать
1. Улучшить index.html — мета-теги, OG, favicon, description
2. Создать SEO-лендинги (статические HTML-страницы через React)
3. Сделать /pricing публичной страницей
4. Добавить активное состояние навигации в sidebar
5. Добавить label к формам
6. Добавить 404 страницу
7. Создать главную landing page для неавторизованных
